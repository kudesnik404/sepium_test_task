Общие требования
* Объём: Необходимо сверстать листинг с мини-карточками (заверстать нужно только 1 мини-карточку и продублировать её).
* Адаптив: Верстка должна быть:
    * кроссбраузерной (корректное отображение во всех современных браузерах),
    * адаптивной (десктопная и мобильная версии).
* Стек технологий:
    * Чистый HTML
    * Чистый CSS
    * JS / jQuery
    * Fancybox
    * Любая библиотека для создания слайдера
* Методология:
    * Соблюдение принципов БЭМ
    * Соответствие макету (Pixel Perfect)

Требования к реализации
1. Слайдер
Картинки в карточке формируют цикличный слайдер.
На десктопе:
* Переключение с помощью кнопок и пагинации
* Без автопроигрывания
На мобильных устройствах:
* Кнопки переключения скрываются
* Пагинация остаётся
* Активируется автопроигрывание (каждые 3 секунды)

2. Название товара
* Длинное название обрезается многоточием
* Может занимать максимум 2 строки

3. Описание товара
* Изначально скрыто
* Появляется при наведении (только на десктопе)
* На мобильной версии остаётся скрытым

4. Модификации
* Активная (выбранная) модификация имеет собственный стиль
* Активной может быть только одна модификация

5. Статичные элементы
* Артикул
* Блок с ценой

6. Кнопка «Купить»
* На десктопе:
    * Изначально скрыта
    * Появляется при наведении
* На мобильной версии:
    * Отображается сразу

7. Ссылки
* Ссылка «Рассчитать стоимость»
    * href="/calc"
    * Открывается в новом окне
* При клике на иконку zoom:
    * Картинки из мини-карточки открываются в модальном окне Fancybox

8. Логика клика по карточке
При клике на мини-карточку (кроме интерактивных элементов):
* элементы управления слайдером
* zoom
* like
* «Купить»
* «Рассчитать стоимость»
должна открываться ссылка:

href="/card"

в новой вкладке.

9. Мобильная версия
* Карточки располагаются друг под другом
* Размеры шрифтов адаптированы под мобильные устройства
* Остальные решения — на усмотрение исполнителя

Будет плюсом (необязательно)
* Использование sprite.svg
* Использование rem
* Анимации и hover-эффекты для интерактивных элементов
* Реализация переключения модификаций при клике
* Реализация переключения состояния like:
    * активное (стили на своё усмотрение)
    * неактивное (как в макете)

Запрещается
* Использование HTML-шаблонизаторов
* Использование CSS-препроцессоров
* Использование фреймворков (React, Vue.js и т.д.)
* Самописная (кастомная) реализация слайдера
